ðŸ”¹ What is Minikube?
- Minikube is a local tool that runs a single-node Kubernetes cluster on your machine.
- Great for development and testing.

ðŸ”¹ Minikube Commands
- Start cluster:
    minikube start --driver=hyperv --hyperv-virtual-switch="MinikubeSwitch"
- Delete cluster:
    minikube delete
- Check Minikube status:
    minikube status
- Get Minikube IP:
    minikube ip
- Open service in browser:
    minikube service <service-name>
- Dashboard (UI):
    minikube dashboard

ðŸ”¹ What is a Pod?
- The smallest unit in Kubernetes.
- Can contain one or more containers.
- Exposes ports only inside the cluster.

ðŸ”¹ Pod YAML Example
apiVersion: v1
kind: Pod
metadata:
  name: shoppingkart
  labels:
    app: shoppingkart
spec:
  containers:
  - name: nginx
    image: svixky/shopping:latest
    ports:
    - containerPort: 8000

ðŸ”¹ Service YAML Example (to expose the pod)
apiVersion: v1
kind: Service
metadata:
  name: shoppingkart
spec:
  type: NodePort
  selector:
    app: shoppingkart
  ports:
  - port: 8000
    targetPort: 8000
    nodePort: 30080

ðŸ”¹ Access the App

ðŸ‘‰ Pods
- Create pod from YAML:
    kubectl apply -f pod.yaml
- Get all pods:
    kubectl get pods
- Describe a specific pod:
    kubectl describe pod <pod-name>
- Delete a pod:
    kubectl delete pod <pod-name>
- Logs of a pod:
    kubectl logs <pod-name>
- Exec into a running pod:
    kubectl exec -it <pod-name> -- /bin/bash

ðŸ‘‰ Services
- Get all services:
    kubectl get svc
- Expose a pod through the command:
    kubectl expose pod <pod-name> --type=NodePort --port=8000 --target-port=8000
- Delete a service:
    kubectl delete service <service-name>

ðŸ‘‰ Apply & Delete YAML files
